    Then implement:

    **lib/State.psm1:**
    - `$ErrorActionPreference = "Stop"` at module scope
    - `$script:RegBase = "HKLM:\SOFTWARE\Baraka\Deploy"` (module-scoped, overridable for tests)
    - `Set-RegistryBase` function: allows overriding $script:RegBase (for testing with HKCU)
    - `Get-DeployState` function: accepts -Name, reads from $script:RegBase, returns value or $null
    - `Set-DeployState` function: accepts -Name, -Value, writes to $script:RegBase (creates path if missing with New-Item -Force)
    - `Export-ModuleMember -Function Get-DeployState, Set-DeployState, Set-RegistryBase`

    **lib/Guard.psm1:**
    - `$ErrorActionPreference = "Stop"` at module scope
    - Imports: depends on Log.psm1 (Write-Log) and State.psm1 (Get-DeployState, Set-DeployState) being loaded in the session
    - `Invoke-Step` function: accepts -StepName [string] and -Body [scriptblock]
      1. Ensure registry base path exists (New-Item -Force)
      2. Check if "$StepName-Done" property equals 1 via Get-DeployState; if so, Write-Log INFO "[$StepName] Already complete -- skipping" and return

      4. try { & $Body } catch { Write-Log ERROR "[$StepName] Failed: $($_.Exception.Message)"; throw }
      5. Set-DeployState -Name "$StepName-Done" -Value 1
      6. Write-Log INFO "[$StepName] Done"
    - `Export-ModuleMember -Function Invoke-Step`

    Follow RESEARCH.md Pattern 3 exactly, but route through State.psm1 helpers rather than raw registry calls. Add try/catch around body execution so failures are logged before re-throwing (CORE-01).
